Hex Viewer - What is it?

A hex viewer is a program which displays data as hexadecimal numbers. It reads files as bytes, and then displays those bytes to the user as hexadecimal numbers. Hex viewers and hex editors are often used for displaying binary data such as compiled code, as they show a clear image of where each byte is in memory, as well as what value it has.

My hex viewer can read bytes from files as well as bytes generated through javascript code, and it will update the page live to reflect those changes as soon as they happen. It also supports highlighting data values, although controlling this functionality is currently not possible through the web interface and has to be done through the javascript console. You can drag and drop your own files into the hex editor (or use the upload button, your choice) to load them and display them as binary data. I do recommend to keep the files well under under 1 megabyte however, as my hex viewer isn't well optimised for large files, and your web browser will likely run out of memory.

How to use it

To use the hex viewer you first need to open a file. You can either choose one of the examples or upload your own.
To upload your own file just drag the file over the browser window and release your mouse button. The hex viewer should display an overlay telling you that you can safely drop the file when you hover a valid file over it. Note that dragging files directly from programs might not always work, and you might have to drag the file over from a folder on your disk.
If you can't or don't want to drag a file over a window, you can use the Browse Files button to select your file.

The hex viewer stores addresses of bytes as hexadecimal numbers. The least significant hex digit is shown on the top bar, while the rest of the digits is shown to the left of the data.

Available Examples

Hex viewer has several example files provided with it, and some examples also have extra functionality provided through javascript. Here is a list of all examples:

  • Incrementing numbers - 256 bytes of unique data in increasing order generated with javascript.
  • Prime numbers - Same as Incrementing numbers, but javascript also highlights prime numbers.
  • Changing numbers - 256 zero bytes. Javascript sets a random byte to a random value every 0.5 seconds.
  • NBT Data (test.nbt) - A test file in the NBT data format. The NBT data format is a binary data format developed by Mojang for their game Minecraft. Find out more on wiki.vg or on Minecraft Wiki.
  • This page (*) - Source code of the hex viewer website. The hex viewer will interpret the text as bytes and display them.

Console commands

The hex viewer stores all of the currently displayed data in the hex object. The hex object works just like a any other javascript array, and all the rendering is done through it. This means that the hex object has extra properties and methods which give you access to the hex viewer's extra functionality.

highlight

hex.highlight(index [, count], style)
This method is used for highlighting pieces of data. It will add the CSS class given in the style parameter to the cell at index. If count is specified it will apply this class to count cells starting at index.

un-highlight

hex.unhighlight(index [, count], style)
This method is used for removing the highlighting from pieces of data. It will remove the CSS class given in the style parameter to the cell at index. If count is specified it will remove this class from count cells starting at index.

CSS classes

Here is a list of provided CSS classes and colors you can highlight your data with:

  • red-highlight - #FF0000 red at 75% opacity
  • green-highlight - #00FF00 green at 75% opacity
  • blue-highlight - #0000FF blue at 75% opacity
