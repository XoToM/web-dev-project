3D Rendering engine - What does it do?

A rendering engine's job is to draw graphics to the screen for the user to see. There are many different types of rendering engines, but a 3D Rendering engine is responsible for drawing 3D graphics. These 3D engines are used for anything from games and movies, to drawing graphs.
My 3D engine was originally designed for drawing graphics for games, however it was repurposed as a model viewing program. It can load 3D models in the GLTF format and render them to the screen. It also gives the user extra tools for playing around with the rendered models like lights and animations.

How to use it

My 3D engine draws things in the following order: 3D graphics, overlay with basic informations, and windows for managing the 3D environment.

Available Examples

Hex viewer has several example files provided with it, and some examples also have extra functionality provided through javascript. Here is a list of all examples:

  • Incrementing numbers - 256 bytes of unique data in increasing order generated with javascript.
  • Prime numbers - Same as Incrementing numbers, but javascript also highlights prime numbers.
  • Changing numbers - 256 zero bytes. Javascript sets a random byte to a random value every 0.5 seconds.
  • NBT Data (test.nbt) - A test file in the NBT data format. The NBT data format is a binary data format developed by Mojang for their game Minecraft. Find out more here or here.
  • This page (*) - Source code of the hex viewer website. The hex viewer will interpret the text as bytes and display them.
